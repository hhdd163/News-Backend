# 头条新闻后端项目 - 完整架构解析

## 📖 项目概述

这是一个基于 **FastAPI** 框架构建的头条新闻后端系统，提供新闻浏览、用户管理、收藏和历史记录等功能。项目采用现代化的异步编程模型，使用 MySQL 作为主数据库，Redis 作为缓存层，实现了高性能的 RESTful API。

## 🔧 核心技术栈

### 1. FastAPI - Web框架

**特点：**
- 现代、高性能的 Python Web 框架
- 自动生成交互式 API 文档（Swagger UI）
- 基于类型提示的数据验证
- 原生支持异步编程（async/await）

**在本项目中的应用：**
```python
# 创建应用实例
app = FastAPI()

# 定义路由
@router.get("/categories")
async def get_categories(...):
    ...

# 依赖注入
db: AsyncSession = Depends(get_db)
```

### 2. SQLAlchemy (Async) - ORM框架

**特点：**
- Python 最流行的 ORM 框架
- 2.0 版本支持异步操作
- 类型安全的查询构建
- 自动会话管理

**在本项目中的应用：**
```python
# 定义模型
class News(Base):
    __tablename__ = "news"
    id: Mapped[int] = mapped_column(Integer, primary_key=True)
    title: Mapped[str] = mapped_column(String(255))

# 异步查询
stmt = select(News).where(News.category_id == category_id)
result = await db.execute(stmt)
```

### 3. MySQL + aiomysql - 数据库

**特点：**
- 关系型数据库，数据结构清晰
- aiomysql 提供异步驱动
- 连接池管理，提高并发性能

**数据库表设计：**
- `user` - 用户信息表
- `user_token` - 用户令牌表
- `news_category` - 新闻分类表
- `news` - 新闻内容表
- `favorite` - 收藏关系表
- `history` - 浏览历史表

### 4. Redis (async) - 缓存系统

**特点：**
- 内存数据库，读写速度极快
- 降低数据库压力
- 支持多种数据结构

**缓存策略：**
```python
# 新闻分类缓存：7200秒（2小时）
CATEGORIES_KEY = "news:categories"

# 新闻列表缓存：1800秒（30分钟）
NEWS_LIST_PREFIX = "news:list:{category_id}:{page}:{size}"
```

### 5. Pydantic - 数据验证

**特点：**
- 基于类型提示的数据验证
- 自动序列化和反序列化
- 与 FastAPI 深度集成

**在本项目中的应用：**
```python
# 定义请求/响应模型
class UserRequest(BaseModel):
    username: str
    password: str

class UserInfoResponse(BaseModel):
    id: int
    username: str
    model_config = ConfigDict(from_attributes=True)
```

### 6. Passlib + bcrypt - 密码加密

**特点：**
- 安全的密码哈希算法
- 防止彩虹表攻击
- 自动加盐处理

**使用方式：**
```python
# 加密密码
hashed_password = pwd_context.hash(password)

# 验证密码
is_valid = pwd_context.verify(plain_password, hashed_password)
```

## 📡 API 接口详解

### 1. 用户模块 (`/api/user`)

#### 1.1 用户注册
- **接口**: `POST /api/user/register`
- **功能**: 创建新用户并返回 Token
- **流程**: 
  1. 接收用户名和密码
  2. 密码加密（bcrypt）
  3. 创建用户记录
  4. 生成 UUID Token（有效期7天）
  5. 返回 Token 和用户信息

#### 1.2 用户登录
- **接口**: `POST /api/user/login`
- **功能**: 验证用户凭证并返回 Token
- **流程**:
  1. 查询用户是否存在
  2. 验证密码（bcrypt比对）
  3. 生成/更新 Token
  4. 返回 Token 和用户信息

#### 1.3 获取用户信息
- **接口**: `GET /api/user/info`
- **认证**: 需要 Token
- **功能**: 获取当前登录用户的详细信息

#### 1.4 修改用户信息
- **接口**: `PUT /api/user/update`
- **认证**: 需要 Token
- **功能**: 更新昵称、头像、性别、简介、手机号

#### 1.5 修改密码
- **接口**: `PUT /api/user/password`
- **认证**: 需要 Token
- **功能**: 验证旧密码并设置新密码

### 2. 新闻模块 (`/api/news`)

#### 2.1 获取新闻分类
- **接口**: `GET /api/news/categories`
- **缓存**: 优先从 Redis 读取
- **功能**: 返回所有新闻分类列表

#### 2.2 获取新闻列表
- **接口**: `GET /api/news/list`
- **参数**: 
  - `categoryId`: 分类ID（必填）
  - `page`: 页码（默认1）
  - `pageSize`: 每页数量（默认10）
- **缓存**: 按分类+页码缓存
- **功能**: 返回分页的新闻列表、总数、是否有更多

#### 2.3 获取新闻详情
- **接口**: `GET /api/news/detail`
- **参数**: `id`: 新闻ID（必填）
- **功能**: 
  - 返回新闻完整内容
  - 自动增加浏览量（views + 1）
  - 返回相关新闻推荐（同分类，按浏览量和时间排序）

### 3. 收藏模块 (`/api/favorite`)

#### 3.1 检查收藏状态
- **接口**: `GET /api/favorite/check`
- **认证**: 需要 Token
- **参数**: `newsId`: 新闻ID
- **功能**: 判断当前用户是否已收藏该新闻

#### 3.2 添加收藏
- **接口**: `POST /api/favorite/add`
- **认证**: 需要 Token
- **功能**: 将新闻添加到用户收藏列表

#### 3.3 取消收藏
- **接口**: `DELETE /api/favorite/remove`
- **认证**: 需要 Token
- **参数**: `newsId`: 新闻ID
- **功能**: 从收藏列表中移除指定新闻

#### 3.4 获取收藏列表
- **接口**: `GET /api/favorite/list`
- **认证**: 需要 Token
- **参数**: 
  - `page`: 页码
  - `pageSize`: 每页数量
- **功能**: 返回用户的收藏列表（联表查询，包含收藏时间）

#### 3.5 清空收藏
- **接口**: `DELETE /api/favorite/clear`
- **认证**: 需要 Token
- **功能**: 删除用户的所有收藏记录

### 4. 历史模块 (`/api/history`)

#### 4.1 添加浏览历史
- **接口**: `POST /api/history/add`
- **认证**: 需要 Token
- **功能**: 记录用户浏览过的新闻（部分实现）

## 🔐 身份认证机制

### Token 认证流程

```
1. 用户注册/登录
   ↓
2. 服务器生成 UUID Token
   ↓
3. Token 存储到 user_token 表（含过期时间）
   ↓
4. 返回 Token 给客户端
   ↓
5. 客户端在请求头携带: Authorization: Bearer <token>
   ↓
6. 服务器验证 Token（查询数据库 + 检查过期时间）
   ↓
7. 验证通过，返回用户对象；失败则返回401错误
```

### 依赖注入实现

```python
# 定义认证函数
async def get_current_user(
    authorization: str = Header(...),
    db: AsyncSession = Depends(get_db)
):
    token = authorization.replace("Bearer ", "")
    user = await users.get_user_by_token(db, token)
    if not user:
        raise HTTPException(status_code=401, detail="无效令牌")
    return user

# 在路由中使用
@router.get("/protected")
async def protected_route(user: User = Depends(get_current_user)):
    # 只有携带有效 Token 的请求才能访问
    pass
```

## 💾 数据库设计

### 核心表结构

#### 1. user（用户表）
```sql
CREATE TABLE user (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,  -- bcrypt加密
    nickname VARCHAR(50),
    avatar VARCHAR(255) DEFAULT '...',
    gender ENUM('male', 'female', 'unknown') DEFAULT 'unknown',
    bio VARCHAR(500) DEFAULT '这个人很懒，什么都没留下',
    phone VARCHAR(20) UNIQUE,
    created_at DATETIME DEFAULT NOW(),
    updated_at DATETIME DEFAULT NOW() ON UPDATE NOW(),
    INDEX username_UNIQUE (username),
    INDEX phone_UNIQUE (phone)
);
```

#### 2. user_token（令牌表）
```sql
CREATE TABLE user_token (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL,
    token VARCHAR(255) UNIQUE NOT NULL,
    expires_at DATETIME NOT NULL,
    created_at DATETIME DEFAULT NOW(),
    FOREIGN KEY (user_id) REFERENCES user(id),
    INDEX token_UNIQUE (token),
    INDEX fk_user_token_user_idx (user_id)
);
```

#### 3. news_category（新闻分类表）
```sql
CREATE TABLE news_category (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) UNIQUE NOT NULL,
    sort_order INT DEFAULT 0,
    created_at DATETIME DEFAULT NOW(),
    updated_at DATETIME DEFAULT NOW() ON UPDATE NOW()
);
```

#### 4. news（新闻表）
```sql
CREATE TABLE news (
    id INT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(255) NOT NULL,
    description VARCHAR(500),
    content TEXT NOT NULL,
    image VARCHAR(255),
    author VARCHAR(50),
    category_id INT NOT NULL,
    views INT DEFAULT 0,
    publish_time DATETIME DEFAULT NOW(),
    created_at DATETIME DEFAULT NOW(),
    updated_at DATETIME DEFAULT NOW() ON UPDATE NOW(),
    FOREIGN KEY (category_id) REFERENCES news_category(id),
    INDEX fk_news_category_idx (category_id),
    INDEX idx_publish_time (publish_time)
);
```

#### 5. favorite（收藏表）
```sql
CREATE TABLE favorite (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL,
    news_id INT NOT NULL,
    created_at DATETIME DEFAULT UTC_TIMESTAMP(),
    FOREIGN KEY (user_id) REFERENCES user(id),
    FOREIGN KEY (news_id) REFERENCES news(id),
    UNIQUE KEY user_news_unique (user_id, news_id),
    INDEX fk_favorite_user_idx (user_id),
    INDEX fk_favorite_news_idx (news_id)
);
```

#### 6. history（浏览历史表）
```sql
CREATE TABLE history (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL,
    news_id INT NOT NULL,
    view_time DATETIME DEFAULT UTC_TIMESTAMP(),
    FOREIGN KEY (user_id) REFERENCES user(id),
    FOREIGN KEY (news_id) REFERENCES news(id),
    UNIQUE KEY user_news_unique (user_id, news_id),
    INDEX fk_favorite_user_idx (user_id),
    INDEX fk_favorite_news_idx (news_id)
);
```

## 🚀 缓存策略

### Redis 键命名规范

```
news:categories                              # 新闻分类（Hash/List）
news:list:{category_id}:{page}:{size}        # 新闻列表（List）
```

### 缓存过期时间设计

| 数据类型 | 过期时间 | 说明 |
|---------|---------|------|
| 新闻分类 | 7200秒（2小时） | 分类数据稳定，缓存时间长 |
| 新闻列表 | 1800秒（30分钟） | 列表数据较稳定，中等缓存时间 |
| 验证码 | 120秒（2分钟） | 临时数据，快速过期 |

**设计原则：**
- 数据越稳定，缓存时间越长
- 避免所有 Key 同时过期（缓存雪崩）
- 不同业务设置不同的过期时间

### 缓存工作流程

```python
# 1. 尝试从缓存读取
cached_data = await get_cached_news_list(category_id, page, size)
if cached_data:
    return [News(**item) for item in cached_data]

# 2. 缓存未命中，查询数据库
news_list = await db.query(News)...

# 3. 写入缓存
news_data = [model_validate(item).model_dump() for item in news_list]
await set_cached_news_list(category_id, page, size, news_data)

# 4. 返回数据
return news_list
```

## ⚠️ 异常处理机制

### 全局异常处理器

项目注册了4层异常处理器，按优先级从高到低：

1. **HTTPException** - 业务逻辑主动抛出的异常
   - 示例：用户不存在、密码错误
   - 返回：对应的 HTTP 状态码和错误信息

2. **IntegrityError** - 数据库完整性约束错误
   - 示例：用户名重复、外键约束失败
   - 智能识别错误类型，返回友好提示

3. **SQLAlchemyError** - 数据库操作错误
   - 示例：连接失败、查询语法错误
   - 开发模式返回详细堆栈，生产模式返回通用提示

4. **Exception** - 兜底处理器，捕获所有未处理异常
   - 确保任何错误都能返回标准化响应

### 统一响应格式

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { ... }
}
```

**错误响应示例：**
```json
{
  "code": 401,
  "message": "无效的令牌或已经过期的令牌",
  "data": null
}
```
